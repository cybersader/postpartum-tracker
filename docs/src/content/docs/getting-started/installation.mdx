## Requirements

- Obsidian 0.15.0 or later
- Works on desktop (Windows, macOS, Linux) and mobile (iOS, Android)

## Install via BRAT (recommended)

[BRAT](https://github.com/TfTHacker/obsidian42-brat) is a community plugin that lets you install beta plugins directly from GitHub.

<Steps>

1. Install **BRAT** from Settings > Community Plugins > Browse.

2. Open BRAT settings and click **Add beta plugin**.

3. Enter `cybersader/postpartum-tracker` and click **Add plugin**.

4. Go to Settings > Community Plugins, find **Postpartum tracker**, and enable it.

</Steps>

BRAT will automatically check for updates when you open Obsidian.

## Shared vaults (Obsidian Sync)

:::tip[Multi-user setup]
If you share your vault with others via **Obsidian Sync**, only **one person** needs to install the plugin via BRAT. To make this work:

1. Go to **Settings > Sync > Vault configuration sync**.
2. Enable **Installed community plugins**.
3. Enable **Active community plugins**.

Once synced, the plugin and its settings will appear on all connected devices. Other users just need to restart Obsidian or toggle restricted mode off and on.
:::

:::caution[Sensitive settings sync]
Plugin settings sync to all users. This includes your **Todoist API token** and **webhook URLs**. If users need separate Todoist accounts, each device should configure its own Todoist settings after syncing.
:::

## Manual installation

<Steps>

1. Download `main.js`, `manifest.json`, and `styles.css` from the [latest release](https://github.com/cybersader/postpartum-tracker/releases).

2. Create the folder `<your vault>/.obsidian/plugins/obsidian-postpartum-tracker/`.

3. Copy the three downloaded files into that folder.

4. Open Obsidian, go to Settings > Community Plugins, and enable **Postpartum tracker**.

</Steps>

## Verify it works

After enabling the plugin:

1. Open any note.
2. Run the command **Insert postpartum tracker** from the command palette (Ctrl/Cmd+P), or click the baby bottle icon in the ribbon.
3. A tracker widget should appear in your note.

If nothing appears, check the developer console (Ctrl+Shift+I on desktop) for errors.
